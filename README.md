# kua-calculator

A free web tool that returns a reader's Kua number, East/West group, and the eight personal directions (four favourable, four unfavourable) with plain-English meanings.

The first product of My Feng Shui Home. Top-of-funnel acquisition channel: someone searches "kua number calculator," lands here, gets the answer in one page, and is offered the deeper methodology, plus an optional free account.

## Stack

- **Next.js** (App Router, TypeScript) on **Vercel**.
- **Auth.js v5 (NextAuth)** on the Next.js server. Google OAuth + Resend magic-link email. Sign-in callbacks live on our own domain (`/api/auth/callback/*`), so Google's consent screen and the magic-link URL both read `myfengshuihome.com`.
- **Supabase** for the Postgres database only. The Auth.js tables live in the `next_auth` schema via `@auth/supabase-adapter`; application data (`profiles`, `saved_charts`) lives in `public` with FKs to `next_auth.users`. Access control is enforced in Next.js server code; RLS is off on the public tables.
- The calculation itself is unchanged vanilla JS, reused from Phases 1-2 as a static island. The four scripts in `public/calculator/` are loaded with `next/script`; no rewrite of the calculator logic.

Phase 3 migrated this project off Flask + Render. The thin Flask server is gone. The NextAuth migration replaced Supabase Auth with Auth.js v5.

## Run locally

```bash
npm install
cp .env.example .env.local   # fill in real Auth.js + Supabase keys for live auth
npm run dev
```

Default URL: http://localhost:3000

For a production-equivalent run:

```bash
npm run build && npm run start
```

The calculator, methodology, and embed pages work fully with the placeholder
values in `.env.local`. Only sign-in / account need real keys.

## Environment variables

Seven variables, documented in `.env.example`:

| Variable | Purpose |
|---|---|
| `AUTH_SECRET` | Auth.js session-cookie signing secret (32-byte base64). |
| `AUTH_GOOGLE_ID` | Google OAuth client ID. Redirect URI: `/api/auth/callback/google`. |
| `AUTH_GOOGLE_SECRET` | Google OAuth client secret. |
| `AUTH_RESEND_KEY` | Resend API key used by the magic-link email. |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL. |
| `SUPABASE_SERVICE_ROLE_KEY` | Service-role key. Server only. Used by the Auth.js Supabase adapter and every server-side DB read/write. |
| `NEXT_PUBLIC_SITE_URL` | Canonical site URL. |

## Database

The current schema is `supabase/migrations/0002_nextauth.sql`. It creates
the `next_auth` schema (the four Auth.js adapter tables: `users`,
`accounts`, `sessions`, `verification_tokens`), recreates the application
tables (`public.profiles`, `public.saved_charts`) with FKs to
`next_auth.users`, and adds a trigger that creates a `profiles` row when
a new NextAuth user is inserted. Row-Level Security is off on the
application tables; access control happens in Next.js server code, which
always knows the NextAuth session user and writes through the
service-role client.

Apply `0002_nextauth.sql` in the Supabase SQL editor (or via the
Supabase CLI). It drops the old `public.profiles` / `public.saved_charts`
and the `handle_new_user` trigger before recreating them, so re-running
it is safe on a fresh project. The original `0001_init.sql` is kept in
the repo for historical reference only.

## Embed it

```html
<iframe
  src="https://your-host/embed"
  width="100%"
  height="900"
  style="border:0;max-width:720px;"
  loading="lazy"
  title="Kua number calculator"
></iframe>
```

`/embed` sends `Content-Security-Policy: frame-ancestors *`; every other route
sends `frame-ancestors 'self'` so the account pages cannot be framed.

## Design

- Palette: warm naturals - cream `#f1e9d8`, sand `#e0d3b8`, deep olive green `#4f5a36`, clay `#be6b43`, deep ink `#2a271e`. Full system in `brand/BRAND_BOOK.md`.
- Typeface: Hanken Grotesk throughout, with Bilbo Swash Caps for the "My Feng Shui Home" signature. Loaded via `next/font/google`.
- No tracking, no analytics on the calculator core. The account layer uses Auth.js + Supabase by design.
